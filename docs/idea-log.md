# Initial IT Venture Idea Log - Team FoodFast

## Idea 1: FoodFast (Campus Pre-Ordering Web App)
### Problem Area
Students face long cafeteria queues during brief class intervals, causing them to rush or skip meals, while vendors lose business due to chaotic manual tracking.

### Target Users
Campus students, faculty staff, and local cafeteria food stall vendors.

### Current Alternative
Students wait physically in line or text/call vendors directly, which causes order mix-ups during peak hours.

### Proposed IT Solution
A lightweight, mobile-responsive pre-ordering platform where students order ahead, pay, and receive text/app alerts when pick-up is ready.

### Possible Technology
Frontend with HTML/CSS/JavaScript and a simple cloud database (like Firebase or Google Sheets API) to track live orders.

### Why This is Suitable
Our team features an eager frontend developer (Arkar) and a structured tester (Thant Zin) which makes a responsive web layout highly achievable over 14 labs.

## Idea 2: QuickPrint (Automated Campus Document Kiosk App)
### Problem Area
The university library printing queues peak right before major assignment deadlines, making students late to turn in paper assignments.

### Target Users
Students needing rapid document printing and campus IT support teams.

### Current Alternative
Waiting in line at the library service desk or utilizing old, coins-only print machines.

### Proposed IT Solution
A cloud-based queue web app where students upload PDFs from their phone, get a unique QR code, and scan it at an automated kiosk to instant-print.

### Possible Technology
Web application with PDF processing scripts and QR code generation libraries.

### Why This is Suitable
Focuses heavily on UI layout design (Mi Su Zin Swar) and simple file upload code logic, making it easy to mock up a working prototype.

## Idea 3: SmartSlot (Campus Study Room Booking Tracker)
### Problem Area
Group study rooms are continuously occupied, and students walk all over campus trying to find empty spaces because there is no live tracking tool.

### Target Users
University group study teams and university facility administrators.

### Current Alternative
Walking physically room-to-room or booking through clunky legacy university web portals.

### Proposed IT Solution
A mobile-friendly dashboard tracker showing live "occupied vs free" statuses of study rooms with quick 1-hour reservations.

### Possible Technology
JavaScript tracking dashboard connected to a shared team spreadsheet backend database.

### Why This is Suitable
Leverages spreadsheet expertise (Mi Su Zin Swar) and basic HTML structures to build a data-driven prototype efficiently.
